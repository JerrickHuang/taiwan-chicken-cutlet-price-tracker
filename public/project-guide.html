二、主題說明：為什麼選雞排

雞排是台灣常見的庶民小吃，很多人對它的價格有生活記憶。從早期 30 元、40 元，到近年熱門商圈可能出現 90 元到 120 元，雞排很適合用來觀察民生物價變化。

生活化

雞排是大眾熟悉的商品，使用者容易理解資料意義。

有趨勢

1993 到 2026 年價格變化明顯，適合做圖表與年份滑桿。

能查詢

不同年份、店家與商圈資料可用表格、搜尋與來源說明呈現。

四、技術架構

本專案採用 Express 同時提供靜態前端與 REST API，前端再用 fetch 呼叫 API，資料儲存在 SQLite。

使用者瀏覽器
  ↓
public/index.html + styles.css + app.js
  ↓ fetch API
server.js / Express.js
  ↓
src/database.js / SQLite
前端

HTML、CSS、JavaScript、Canvas、搜尋框、滑桿、表格、即時新增回饋。

後端

Node.js、Express.js、SQLite、REST API、爬蟲示範與 Azure 部署設定。

五、SQLite 資料庫設計

資料表為 price_records，支援單一年份、期間資料、價格區間、來源與推估註記。

欄位用途
record_date使用者輸入日期或資料代表日期
item_name商品名稱，預設為雞排
period顯示用期間，例如 1993-1995
year_start / year_end支援年份滑桿與圖表計算
price_ntd代表價格
price_min_ntd / price_max_ntd價格區間
is_estimated0 為來源紀錄，1 為推估資料
source_ref / source_url來源代號與來源網址
Azure 部署時資料庫路徑會自動改成 /home/data/data.db，避免每次重新部署就清掉雲端資料。

六、Express API 規格

Method路徑功能
GET/api/health檢查執行環境與資料庫路徑
GET/api/prices取得價格資料，可用 q、year、estimated 篩選
POST/api/prices新增一筆價格紀錄
GET/api/prices/summary取得資料筆數、最高價、最低價、平均價
GET/api/sources取得來源列表
POST/api/crawl爬蟲示範，讀取公開來源網頁標題與狀態碼
 
{
  "record_date": "2026-06-20",
  "item_name": "雞排",
  "price_ntd": "120"
}

七、OpenSpec / SDD 流程

本專案不是直接寫程式，而是先用 OpenSpec 定義需求、設計與任務，確定網站要做什麼後再進入實作。

文件用途
proposal.md說明專案原因、功能變更與影響範圍
design.md說明技術架構、資料模型、API 與部署設計
tasks.md將專案拆成可檢查的任務清單
specs/細分價格管理、查詢、視覺化、爬蟲、部署文件規格
